CONFIRMED:
- Guest VMPL = 0 (read from /sys/devices/system/cpu/sev/vmpl).
  snpguest --vmpl 0 succeeds → no SVSM hierarchy. The earlier "VMPL 1"
  in the report was just snpguest's default (--vmpl defaults to 1).
- No vTPM (`ima: No TPM chip found, activating TPM-bypass!`). systemd's
  TPM-PCR services all skipped with ConditionSecurity=measured-uki unmet.
- No SVSM in dmesg. SVSM_CAA section in OVMF is just OVMF being
  SVSM-compatible; it is NOT active.
- VPSBG uses `-kernel`-style boot path (linuxboot_dma.bin in fw_cfg
  bootorder; VMMBootOrder0000 EFI variable set by VMM).
- Determinism: 3 reports with random REPORT_DATA all yield identical
  MEASUREMENT, HOST_DATA, ID_KEY_DIGEST, AUTH_KEY_DIGEST — chip is
  deterministic across requests, as expected.

NEW DATA POINT:
- Current Slice 2 MEASUREMENT (rebuilt revert UKI bpir-slice2-revert.efi):
    d7dad68c9b0dcb40de6f45b8d11bfed95f9f56da9fc1b353b794a7d187faa2b7800b1792ad8e0e38e1afc21ee767a906
  Differs from the original Slice 2 baseline e522983f... because the
  UKI was rebuilt during Slice 3 dev (different mtimes leak into cpio).

REMAINING UNKNOWN:
- How the UKI ends up in measured memory. The simple "QEMU -kernel
  pre-loads kernel as NORMAL pages at address X" model fails for
  every X tried (1 MB-aligned 1 MB to 4 GB, plus 4 KB-aligned near
  the standard 1 MB / 16 MB / 256 MB Linux kernel load addresses).

OUTSTANDING HYPOTHESES (in order of remaining plausibility):
1. QEMU loads kernel at a non-standard address (off-by-N from
   conventional spots). Brute-force at 4 KB granularity over wider
   range possible but already exhaustive at 1 MB granularity.
2. UKI loaded into measured memory in chunks (PE sections separately,
   as EFI loaded image). Multi-region search infeasible without hint.
3. Custom QEMU `args:` injection by VPSBG portal that doesn't fit
   any standard mechanism.
4. IGVM file describes the launch (Proxmox VE 9 supports this). With
   IGVM, addresses + page types are arbitrary per-guest.

NEXT STEP: VPSBG support email asking for either the IGVM file, the
